## D5 Implementation Check

- `StoryGraphPanel` renders DS `PassageNode`s from `queryGraphProjection()`.
- Viewport/focus options are passed through the query boundary.
- Generated layout save flows through the host command path.
- Migrated editor controls for passage create/delete/move/rename/start,
  story rename, story format, snap-to-grid, zoom, text, script, stylesheet, and
  asset operations flow through `CoreProjectHost.applyStoryCommand()`.
- Play/Test preview routes no longer reload on story-format load-state churn, and
  in-flight story-format property loads dedupe by format ID.
- Explicit reveal-in-graph is wired from Text mode, search selection, and
  diagnostics. When invoked from Text mode, the workspace switches to Split so
  the selected passage is visible in the graph while the source context remains
  available.
